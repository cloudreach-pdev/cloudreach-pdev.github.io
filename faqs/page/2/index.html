Frequently Asked Questions

How can I be sure my data is kept separate?

As a multi-service provider capable Cloud ITSM, Conductor does not have direct affinity for any specific cloud vendor or provider, and realistically any API capable service can be integrated easily. As of launch, Conductor offers discovery solutions for AWS, which will be augmented shortly with support for Azure and GCP.

How can I define my application within Conductor?

Conductor’s resource group model is open ended and can be used for multiple purposes, including ‘representation’ of an application within an Organisation. Provided a query can be used to define what the application is (e.g. everything tagged application:MyApplication), then Conductor can represent this for you. This makes it easy to apply application specific policies to your applications as needed.

How do I actually procure Conductor?

Conductor is currently open enrolment meaning you can sign-up freely with nothing more than an e-mail address (or social identity such as Google Apps or LinkedIn) and an organisation name. We will get in contact via email within 24 hours with instructions on how to get started.

How do I know or change my provisioned storage capacity?

Your storage capacity is measured every 24 hours and displayed on a per organisation basis along with the current capacity tier procured. This can be changed (up-or-down) by getting in touch with your account manager or raising a support case. In the near future this will be modifiable via the settings page of Conductor. For managed service providers using Conductor across multiple organisations, please raise a support case and we will get in touch to quota capacity across organisations you specify.

How is data isolated between customers?

Customers’ data is separated at rest by associating all data in our database with an “Organisation” key. This is unique to a customer. Our APIs also segregate data and validate all data returned is associated to the same Organisation key. We audit our public and private APIs for compliance to this requirement with every build / release. See our privacy policy and security policy for a detailed breakdown of our respective postures.

If I am using this as an MSP, can I query across Orgs?

Due to restrictions in place to prevent cross-organisational contamination or compromise, it is not possible to query universally across all organisations which you may be managing (under an MSP agreement with Cloudreach). It is however possible to enumerate a query across all organisations (i.e. the same query is made against all organisations) and results aggregated.

What cloud providers do you support?

As a multi-service provider capable Cloud ITSM, Conductor does not have direct affinity for any specific cloud vendor or provider, and realistically any API capable service can be integrated easily. As of launch, Conductor offers discovery solutions for AWS, which will be augmented shortly with support for Azure and GCP.

What size customer is Conductor aimed for?

Conductor is primarily aimed at cloud enabled enterprises with complicated operational methodologies and requirements. This however does not preclude SME’s from using Conductor, in fact many smaller organisations can benefit from our free tier plan to get started. Additionally, Conductor is able to be used by managed service providers who may manage multiple other organisations, there are several ways of achieving this effectively, so please get in touch.

Where can I see the solutions it supports now?

Conductor has an evolving eco-system of solutions which are offered directly via its solution gallery. The solutions and code can be found in our GitHub organisation. Please note that Cloudreach developed global solutions are open source and can be extended freely by customers of the Conductor platform.

Why is the cost model for Conductor storage based?

Conductor uses a pay-per-provisioned GB model due to the ease of estimating costs to users and because it is able to be directly correlated to value for customers. Solutions offered as global via the included solution gallery will remain free, however this does not prevent third-party or even additional solutions from being offered on a paid-for basis in the future.
