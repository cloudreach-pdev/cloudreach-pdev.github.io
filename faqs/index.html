Frequently Asked Questions

Can Conductor handle high-velocity stream data?

In its original incarnation, Conductor can support only low velocity stream data via an API. The majority of current edition solutions use poll-type patterns for ingesting data. The roadmap for the next 6 months includes support for stream-based data patterns, and customers notified of this when available for early-access or general availability. We are always happy to aid any development efforts, so please let us know if we can help.

Can I authenticate with Conductor via my own Identity Provider?

Yes! Conductor supports a Bring-Your-Own-IdP model, meaning we will help you integrate an enterprise identity provider (e.g. Azure AD, ADFS, GApps etc.) which you can then map to users and permissions. Until self-service options become available, the easiest way to get started with this is to raise a support case, and we can help you from there. Until an IdP is configured, local credentials can be issued via the platform for you and your users.

Can I export data to another format or system?

Conductor supports exporting all Organisational data via JSON or CSV. This includes not only resource data, but supporting information around resource groups, solutions and policies in place. This data can be post-processed as needed or stored for compliance reasons. The exported data format is always versioned. It should be noted that it is possible to export data returned from a query in these formats also for reporting purposes.

Can I extend the Conductor UI?

Solutions are used to extend functionality in Conductor and can make use of available UI components. As of the present, these components are developed as part of the core platform and whilst can be leveraged by developed solutions, are unable to themselves be extended by developers. If this is a requirement, please get in touch.

Can I have my cloud assets fully managed by Conductor, including break/fix?

Conductor provides some solutions which happen to include automatic remediation for some cases. These are heavily dependent on the solutions themselves, and users are able to easily extend them as required to meet a broad range of requirements. Cloudreach offers service support contracts for AWS, Azure and GCP hosted applications and infrastructure via a their Cloud Core offering which includes use of Conductor. Please get in touch for more information if this is of interest.

Can I host my own version or appliance of Conductor?

At this time Conductor is offered on a SaaS model basis only. Depending on demand it will be possible to deploy an appliance or instance of Conductor in Kubernetes. If this is of genuine requirement to you or your organisation, please get in touch using email / Slack details found at the bottom of this page under Questions / Support, and we can discuss this.

Can I integrate this with my build / CI platform or system?

In a word, yes. Conductor has 100% RESTful API coverage which can be used by other services. A full reference to this can be found here. Conductor support includes helping development efforts, so get in touch if you need help or guidance integrating your services.

Conductor mentions queries a lot, does this use a defined query syntax?

We use a modified version of Apache Lucene for our query syntax. This was chosen due to its industry reputation and prevalence. At this time Conductor does not support the full specification for Lucene (reference here), but will continue to evolve over time, directed by market demand. If a particular capability is missing which you would like to see included, please get in touch.

Do you event source resource changes?

Resources are point-in-time records at the moment, meaning that any resource which is updated will lose resolution of that change. Keeping a historical record of changes in-line with a retention policy will be available in the next 6 months and allow users to see the full change log set for a given resource. Until this feature is available, it should be noted that Conductor can emit events when a resource changes, along with a record of that change. This can be used to create a ‘dumb’ manifest of changes for a resource if required.

Do you support multi-factor authentication?

We will support this within the next 3 months.
