For Developers

There are two aspects to developing with Conductor depending on what you need to achieve. Where an organisation may wish to integrate the Conductor platform, an API is made available which can be used to achieve anything possible via the WebUI or CLI. If developing new or extending existing solutions is more in-scope, the SDK and Conductor development lifecycle can be used to quickly prototype, test and deploy quality solutions into the Solution Gallery for your organisation.

  • Integrating the Conductor Platform

    You can use or access Conductor freely from an external service, system or automated process by simply enabling an API key for you Organisation through the WebUI or CLI. This follows a similar pattern of access as most other SaaS platforms, for example GitHub.com. Please see our API reference here to get started.

  • Developing Conductor Solutions

    A large part of the Conductor platform is its extensibility. By using the Conductor SDK, developers can extend or create whatever solutions they wish, without worrying about how a solution needs to be rolled out across providers, how to manage credentials or access critical data to drive automation or processes. Please see our SDK documentation here to get started with your first solution.

  • Requesting Development Support

    We are aware that the development documentation and self-direction is not as mature as the platform today. If you need help with getting started with developing solutions or constructing an integration via the API, we are happy to help. Simply raise a support case below.
